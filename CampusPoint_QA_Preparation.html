Q: Kenapa ada 3 smart contract terpisah?

Jawaban: Prinsip Separation of Concerns: 1. CampusPoint.sol - Fokus token saja (ERC-20) 2. ActivityCertificate.sol - Fokus NFT saja (ERC-721) 3. ActivityManager.sol - Fokus logika bisnis

Keuntungan: Mudah di-upgrade, lebih aman, testing lebih mudah.

Q: Bagaimana keamanan sistem ini?

Jawaban: 1. onlyOwner modifier - Hanya admin yang bisa buat kegiatan/kirim poin 2. require() checks - Validasi input di setiap fungsi 3. Ownership pattern - ActivityManager mengontrol token & NFT 4. Blockchain immutability - Data tidak bisa diubah

Q: Kenapa ownership CampusPoint ditransfer ke ActivityManager?

Jawaban: Token hanya bisa di-mint oleh owner-nya. Agar ActivityManager bisa otomatis mint token saat reward, ownership harus ditransfer dari deployer ke ActivityManager.

Q: Bagaimana jika Ganache di-restart?

Jawaban: - Semua data blockchain hilang (termasuk kontrak yang sudah deploy) - Perlu deploy ulang 3 kontrak - Update alamat di contracts.js - Transfer ownership lagi

Solusi production: Deploy ke testnet (Sepolia) atau mainnet.

Q: Bagaimana verifikasi sertifikat?

Jawaban: 1. Cek tokenURI(tokenId) di blockchain 2. URI mengarah ke metadata di IPFS 3. Metadata berisi info sertifikat + gambar 4. Tidak bisa dipalsukan karena tercatat di blockchain

Q: Kenapa mahasiswa perlu MetaMask?

Jawaban: - Setiap transaksi blockchain butuh signature (tanda tangan digital) - MetaMask menyimpan private key dengan aman - User tetap kontrol penuh atas aset mereka

Q: Bagaimana skalabilitas sistem ini?

Jawaban: Untuk production: 1. Gunakan Layer 2 (Polygon, Arbitrum) - gas fee lebih murah 2. Batch transactions - kirim banyak poin sekaligus 3. Off-chain computation dengan on-chain verification

Alur Pemberian Poin

1. Admin pilih kegiatan & mahasiswa
2. Frontend panggil rewardStudent(activityId, studentAddress)
3. Smart contract cek: apakah kegiatan ada? aktif? alamat valid?

5. Token CPNT dikirim ke wallet mahasiswa
6. Event StudentRewarded di-emit

Alur Penerbitan Sertifikat

1. Admin input data + IPFS URI
2. Frontend panggil mintCertificate(activityId, student, uri)
3. Smart contract panggil ActivityCertificate.mintCertificate()
4. NFT dibuat dengan tokenId unik
5. URI metadata disimpan di blockchain
6. Event CertificateMinted di-emit
